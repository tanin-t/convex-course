# For hands-on setup, make Cursor a step-by-step teacher constrained by the page — not a free-form codegen tool

**Rule.** When a Lesson asks students to do hands-on setup or write code that involves Cursor, the Cursor prompt must (1) instruct Cursor to *read this specific Lesson page* and follow its exercise steps as the authoritative outline, (2) frame Cursor as a *teacher* that explains what each step does and why, (3) hand some sub-steps to the student to type themselves (for retention), and (4) require Cursor to pause for confirmation between steps. Do not ship copy-paste prompts whose output is the answer.

**Why.** Feedback Issue #4 (week-1/session-1, type: confusion). The original Exercise step 4 was *"Deploy function ตัวอย่างที่ Convex ให้มาในโปรเจกต์ (ดูใน folder convex/)"* — but `npx convex dev` doesn't actually scaffold a sample function for students to deploy, so the step had no concrete referent and stuck students. The deeper problem the user surfaced: when a Lesson hands students a generation prompt, each student's AI output differs slightly, so they fall off the same step path, the class drifts, and the teacher can't help everyone in parallel. Anchoring Cursor to the Lesson page as the script keeps the class synchronised; framing Cursor as a teacher (not a code generator) restores learning value to the AI-assisted parts.

**How to apply.**
- For any setup / wiring / "first feature" exercise: put exactly one `<CursorPrompt>` above the `<Exercise>` with text shaped like the teacher prompt in `course/app/week-1/session-1/page.tsx`. The prompt's anchor sentence is *"อ่านหน้า /week-{N}/session-{M} บนเว็บ course นี้ก่อน แล้วทำตัวเป็นครู พาฉันทำ section ทดลองเอง ทีละขั้น"*.
- The prompt's rules block must include: stay on the page's steps, break each into sub-steps, explain *what + why*, hand some sub-steps to the student, wait for "ok/ต่อ" before advancing. Without these, Cursor reverts to bulk-generating.
- Exercise steps then become a *high-level outline* the student reads to know what's coming — not a precise click-by-click recipe (Cursor does the click-by-click expansion). Keep each step one sentence; avoid baking in commands that depend on Convex/Next.js CLI behaviour you haven't double-checked exists.
- Verify each `<Exercise>` step references a thing that *actually exists out of the box*. Step 4's regression — "deploy the sample function Convex gives you" — happened because nobody re-ran the bare `create-next-app` + `npx convex dev` flow before publishing. Re-run the cold start before approving.
- Cursor-as-teacher applies from Session 1 onward. By later weeks (Sessions with real features, not setup) the prompt can shift toward pair-programming framing, but the *anchor-to-this-page, wait-between-steps, hand-some-back* shape is permanent.
